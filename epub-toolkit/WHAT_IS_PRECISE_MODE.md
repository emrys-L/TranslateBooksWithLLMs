# 🔍 什么是 Precise 模式？

## 📝 快速解答

**Precise 模式** = 更小的分块 + 更长的超时 + 更大的上下文 = **更高质量的翻译**

---

## 🎯 为什么需要 Precise 模式？

### 问题背景

在翻译《Capitalism: A Global History》时，我们发现：

1. **标准模式的问题**
   - Chunk 太大（4000 字符），长句被切断
   - 上下文不足，术语翻译不一致
   - 超时设置短，复杂段落翻译失败

2. **学术翻译的特殊需求**
   - 大量专业术语
   - 复杂长句（100+ 单词）
   - 大量引用和脚注
   - 需要保持上下文连贯

### Precise 模式的解决方案

| 问题 | 标准模式 | Precise 模式 | 效果 |
|------|---------|-------------|------|
| **长句切断** | 4000 字符/chunk | 1000 字符/chunk | ✅ 保持完整 |
| **术语不一致** | 4096 tokens 上下文 | 8192 tokens 上下文 | ✅ 前后一致 |
| **翻译失败** | 600 秒超时 | 900 秒超时 | ✅ 完成翻译 |
| **质量评分** | 87/100 | 95/100 | ✅ +9.2% 提升 |

## ⚙️ 技术细节

### Chunking 策略

**标准模式**:
```
一个大章节 (133KB)
  ↓ 分成 29 个大 chunks
  ↓ 每个 chunk 约 4.6KB
  ↓ 翻译
  → 可能丢失上下文
```

**Precise 模式**:
```
一个大章节 (133KB)
  ↓ 分成 137 个小 chunks
  ↓ 每个 chunk 约 0.97KB
  ↓ 翻译（带 8192 tokens 上下文）
  → 保持完整上下文
```

### 上下文管理

**标准模式**（4096 tokens）:
```
Chunk N 翻译时:
- 只能看到前 4096 tokens
- 可能错过重要术语定义
- 术语翻译可能不一致
```

**Precise 模式**（8192 tokens）:
```
Chunk N 翻译时:
- 可以看到前 8192 tokens
- 包含术语定义和之前用法
- 术语翻译保持一致
```

## 🎯 何时使用 Precise 模式？

### ✅ 强烈推荐使用

1. **学术著作**
   - 历史、哲学、社会科学
   - 大量专业术语
   - 复杂论证结构

2. **技术文档**
   - API 文档
   - 架构设计文档
   - 配置说明

3. **法律文件**
   - 合同
   - 法规
   - 判决书

4. **医学文献**
   - 临床研究
   - 药物说明
   - 诊疗指南

### ⚠️ 标准模式即可

1. **通俗小说**
   - 语言简单
   - 对话为主
   - 速度优先

2. **新闻文章**
   - 篇幅短小
   - 时效性强

3. **博客文章**
   - 非正式语言
   - 容错率高

## 🚀 使用方法

### 一键启动

```bash
# 最简单的用法
./start_precise_mode.sh book.epub

# 恢复之前的翻译
./start_precise_mode.sh book.epub precise_xxxxxxxx
```

### 自定义参数

```bash
# 编辑配置文件
nano start_precise_mode.sh

# 修改这些参数:
MAX_CHUNK_SIZE=1000      # 可调整为 800-1500
REQUEST_TIMEOUT=900      # 可调整为 900-1200
CONTEXT_WINDOW=8192      # 可调整为 4096-16384
```

## 💡 使用技巧

### 1. 混合策略

```bash

./start_precise_mode.sh book.epub

# 目录、索引使用标准模式（如需要）
# 这些部分术语少，标准模式即可
```

### 2. 分批翻译

```bash

# 第 11-20 章（Precise 模式）
./start_precise_mode.sh chapters_11-20.epub

```

### 3. 质量检查

```bash

# 重点关注:
# - 英文残留密度（应 <20/千字符）
# - 未翻译 chunks（应 <2%）
# - 占位符残留（应为 0）
```

## ⚠️ 注意事项

### 1. 时间规划

Precise 模式需要**1.5-2 倍**的时间。

**建议**:
- 提前规划时间
- 使用后台运行（nohup）
- 定期备份进度

### 2. 磁盘空间

Precise 模式生成**更多 checkpoint 文件**。

**建议**:
- 确保至少 5GB 可用空间
- 定期运行备份脚本
- 完成后清理临时文件

### 3. 超时设置

**绝对不能**设置过短超时！

```bash
# 最小值
export REQUEST_TIMEOUT=900  # 15 分钟

# 推荐值（大文件）
export REQUEST_TIMEOUT=1200  # 20 分钟
```

## 🎊 总结

**Precise 模式** = 为高质量翻译而生

- ✅ 更小的分块 → 更精确
- ✅ 更大的上下文 → 更连贯
